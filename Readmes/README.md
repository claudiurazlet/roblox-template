Generated by [Rojo](https://github.com/rojo-rbx/rojo) 7.5.1.

## Template Stack Defaults

This template includes several packages in `wally.toml`, but not every package is already integrated into gameplay code.

- UI work should generally start from `React` + `ReactRoblox` under `src/UI`.
- Networking defaults to `Networker`; see `Readmes/networker.md` and the `BuildService` example.
- New player-data work should generally start from `Dataservice` + `ServicePlayerData`.
- `ProfileService` remains available as a server-side fallback or lower-level option when a project needs direct control instead of the higher-level data stack.

When adding new template features, treat these as recommended defaults rather than guarantees that every package is already wired into runtime code.

## Getting Started
To build the place from scratch, use:

```bash
rojo build -o "roblox-template.rbxlx"
```

Next, open `genRojoTree.rbxlx` in Roblox Studio and start the Rojo server:

```bash
rojo serve
```

For more help, check out [the Rojo documentation](https://rojo.space/docs).
